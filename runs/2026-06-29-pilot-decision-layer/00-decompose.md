# פיילוט-פרוסה-אחת — פירוק המסך (מסלול-1, מטרה-תחילה)

> תאריך: 2026-06-29 · מקור-חוזה: `../2026-06-28-screen-anatomy/07-spec-v3.md`
> **מסך-נבחן:** המסך הראשי של שכבת-ההחלטות עצמה (Design OS) — outline (עץ-צמתים) משמאל + inspector במרכז (§6).
> **היקף-עיגון נבחן:** דרגה-3 web בלבד, מבודד (אין מאגר-אישי → דרגה-1-2 = 0 בכוונה).
> **השער הקיומי:** AC-12 / §11.1 — ≥30% שיקולים בדרגה-1-3. כאן (מבודד) זה מצטמצם ל-**≥30% דרגה-3** (web מצוטט אמיתי).

## מטרת-המסך (שורש, מטרה-תחילה)
"לתת למעצב לארגן את ההחלטות של מסך, ולהחזיק רציונל בר-הגנה — בלי שהמערכת תעצב במקומו."

## מכלים (מטרה + בדיקת-הרכב-על-ילדים)
- **A. פאנל-Outline (שמאל)** — מטרה: לתפוס ולנווט את מבנה-ההחלטות של המסך.
- **B. פאנל-Inspector (מרכז)** — מטרה: לעבוד החלטה-של-צומת-בודד.
- **C. כניסה/מסלול** — מטרה: להכניס מסך אל הכלי.

## יחידות-החלטה נבחנות (6 — מרחב-משתנים §5; ספרד-על-פני-צירים, מעורב קל/קשה לעיגון)
| # | יחידת-החלטה | מכל | ציר-עיקרי (§5) | למה נבחרה |
|---|---|---|---|---|
| **DU1** | איך מציגים את מבנה/עומק עץ-הצמתים (indent-tree / breadcrumb / zoom) | A | מיקום/סדר + דפוס | web-עשיר (tree/IA) — מבחן הוגן לצד-הקל |
| **DU2** | איך מסמנים חשיבות/סטטוס של צומת בעץ (צבע/badge/משקל) | A | בולטות | web-עשיר (היררכיה-ויזואלית/salience) |
| **DU3** | איך פורשים את מרחב-המשתנים (7 צירים) בלי קיר-טקסט — חשיפה-הדרגתית | B | התנהגות/מצבים + מידע | web-בינוני (progressive disclosure) |
| **DU4** | איך מציגים מקור+דרגה על כל שיקול (badge/אייקון/צבע) — תצוגת-AC-11 | B | מידע-מוצג + בולטות | web-בינוני-קשה (provenance/trust UI) |
| **DU5** | איך מציגים אלטרנטיבות לכל-ציר (radio / cards / list) | B | התנהגות | web-בינוני (option/comparison UI) |
| **DU6** | איך בוחרים/מחליפים בין 2 המסלולים (מטרה-תחילה / ייבוא-Figma) | C | קיום + דפוס | web-קשה/חדשני (mode-switching) — מבחן לצד-הקשה |

> **למה אלה, ולמה לא רק קלים:** מבחן-הגון לשער דורש גם DU שבהם web עלול **להיכשל** (DU4, DU6). לבחור רק tree/hierarchy (web-עשיר) = לרמות את ה-30%.

## פרוטוקול
1. לכל DU → סוכן-מחקר (`חוקר-רקע`, משמעת-URL קשוחה) מחפש שיקולים מעוגנים-web.
2. כל שיקול: טענה + URL-אמיתי + מה-המקור-אומר + דרגה-עצמית (3=מחקר-מצוטט / 4=מוסכמה / 5=אינטואיציה).
3. **המתזמר מאמת URLs בפועל** לפני דירוג (הסוכן מייצר, המתזמר מאמת — דפוס נעול).
4. דירוג-סופי + פיזור-גולמי (כמה 3/4/5) + חישוב שער ה-≥30%.

## מה זה לא בודק (בכוונה)
- מסלול-2 (ייבוא-Figma) לא רץ — §7 קובע שהוא משני; השער לא תלוי-מסלול.
- דרגה-1-2 (מאגר-אישי) = 0 בכוונה — מבודדים את המנוע-האוטומטי מבעיית-ה-cold-start.
- איכותני (סעיפים 1-3 בהגדרת-בוצע) — נשפט בנפרד אחרי; הפיילוט מתרכז בשער (4) האובייקטיבי.
